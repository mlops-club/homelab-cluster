# Qwen3 8B on RTX 5080 — GuideLLM benchmark (2026-06-11)

**Server**: Ollama 0.30.6, single replica, cluster-node-4 (RTX 5080, 16 GB VRAM)
**Client**: GuideLLM 0.6.0 in-cluster pod, synchronous profile (1 request at a time)
**Scenarios**: 4 runs, ~1738 s total benchmark time

## Caveats

- **Qwen3 thinking mode is on by default** — the model emits a long `<think>` block before its actual answer. Observed median output tokens (~900) are much higher than the requested cap (`output_tokens=128/256`). TPOT/throughput are the right metrics to compare across scenarios; raw latency is dominated by output length.
- **Synchronous profile**: one request at a time. The Ollama deploy uses `OLLAMA_NUM_PARALLEL=1` on the 16 GB VRAM budget, so concurrent benchmarks would just queue — measured RPS equals 1/median-latency.
- **First request in each scenario eats a cold-load cost** if Ollama unloaded the model between scenarios. With `OLLAMA_KEEP_ALIVE=24h` this is usually only for the very first scenario.
