# loseit-agent

FastAPI service that fronts a Kitaru-wrapped pydantic-ai agent driving the [`lose-it` CLI](https://github.com/phitoduck/lose-it). The Open WebUI Pipe function streams chat prompts here, watches tool calls land as live status events, and surfaces the final reply back into the chat. Reachable in-cluster at `http://loseit-agent.loseit-agent.svc.cluster.local` (port 80) and from any Tailnet device at `https://loseit-agent.priv.mlops-club.org` (private Ingress via `traefik-private`, wildcard TLS from the reflector-replicated `priv-wildcard-tls` secret — no public exposure).

> **Slice S2** ships only the wire: `POST /run` is an SSE *echo* (tool → tool_done → final) so we can prove the Pipe ↔ FastAPI plumbing end-to-end before any agent logic lands. Slices S4-S6 replace the echo with the real pydantic-ai + KitaruAgent + loseit stack — see `SPEC.md`.

## Layout

```
apps/loseit-agent/
├── Dockerfile        # uv slim base, non-root, exposes 8000
├── server.py         # FastAPI app (POST /run SSE + GET /healthz)
├── manifest.yaml     # Namespace, Deployment, Service, Ingress
├── deploy.sh         # build + push to Harbor + kubectl apply
├── SPEC.md           # contract: endpoints, wire format, auth
└── ARCHITECTURE.md   # mermaid topology + sequence
```

## Auth

A shared bearer token in the `agent-token` Secret (key `token`) is mounted into the pod as the `AGENT_TOKEN_EXPECTED` env var. Every `POST /run` must include `Authorization: Bearer <token>`. `GET /healthz` is unauthenticated so kubelet probes work. The token is generated by `deploy.sh` on first deploy and NOT rotated on subsequent runs — rotating it would invalidate the Pipe config we land in S3.

## Deploy

```bash
./apps/loseit-agent/deploy.sh
```

Idempotent. Builds an `amd64` image tagged `<short-sha>` (`-dirty` if the tree has uncommitted changes), pushes to `cr.priv.mlops-club.org/loseit-agent/agent`, ensures the namespace + Harbor pull secret + agent-token secret exist, then `envsubst | kubectl apply`s the manifest. Requires `HARBOR_ADMIN_PASSWORD` in repo-root `.env`.

Override the tag if you want to redeploy an older image:

```bash
IMAGE_TAG=abc1234 ./apps/loseit-agent/deploy.sh
```

## Verify

```bash
kubectl -n loseit-agent rollout status deploy/loseit-agent
kubectl -n loseit-agent get ingress loseit-agent
curl -sSf https://loseit-agent.priv.mlops-club.org/healthz
```

End-to-end SSE echo (requires a Tailnet-connected device):

```bash
TOKEN=$(kubectl -n loseit-agent get secret agent-token -o jsonpath='{.data.token}' | base64 -d)

curl -sN -X POST https://loseit-agent.priv.mlops-club.org/run \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"hello world"}'
```

Expected output (three SSE frames, in order):

```
data: {"kind":"tool","name":"echo","args_preview":"hello world","call_id":"<uuid>"}

data: {"kind":"tool_done","call_id":"<same uuid>"}

data: {"kind":"final","text":"echo: hello world"}
```

## Troubleshooting

| Symptom | Likely cause |
|---|---|
| `503` from `/run` with body mentioning `AGENT_TOKEN_EXPECTED` | The `agent-token` Secret didn't mount. Check `kubectl -n loseit-agent describe pod -l app=loseit-agent`. |
| `401` from `/run` | Token mismatch. Re-read it: `kubectl -n loseit-agent get secret agent-token -o jsonpath='{.data.token}' \| base64 -d`. |
| `curl` hangs and no events arrive | Traefik buffering. The server already sets `X-Accel-Buffering: no`; confirm the ingress class is `traefik-private`. |
| `ImagePullBackOff` | Harbor pull secret missing or wrong password. Re-run `deploy.sh` after updating `.env`. |
